# SuperIronRuby architecture

SuperIronRuby is a Ruby 4 implementation on .NET 10. It is organized as three
layers — **parser**, **runtime**, **interpreter** — plus a **builtin library**
and a **hosting/CLI** front end. This document explains how each works and how to
extend them.

## Pipeline

```
Ruby source ──▶ PrismParser ──▶ Prism AST ──▶ Interpreter ──▶ result
                   │                              │
              libprism (P/Invoke)          RubyContext + Builtins + Lib/core/*.rb
```

## 1. Parser (`Src/SuperIronRuby/Parser`)

Ruby is parsed by **Prism**, the official portable Ruby parser (the same one
CRuby 4.0, JRuby and TruffleRuby use), so SuperIronRuby gets the full, exact
Ruby 4 grammar for free.

- **`PrismNative.cs`** — P/Invoke to the vendored `libprism.so` (prism v1.8.1).
  `pm_serialize_parse` produces Prism's binary serialized AST (identical bytes to
  `Prism.dump`). A `NativeLibrary` resolver finds the `.so` next to the assembly.
  `PrismParser.Parse` also has an `SIR_PARSER_BACKEND=mri` fallback that shells
  out to the system `ruby`.
- **`Nodes.g.cs` / `Loader.g.cs`** — generated by `Tools/generate_nodes.rb` from
  `Vendor/prism/config.yml`: 151 node classes (one per Prism node type) and the
  per-node binary reader. Hand-written `Loader.cs` has the primitives (LEB128
  varints, integers→long/BigInteger, the constant pool, locations, diagnostics)
  ported faithfully from prism's `serialize.rb`. **To regenerate:**
  `ruby Tools/generate_nodes.rb`.

A subtle detail worth knowing: `DefNode` serializes an extra `uint32` after its
location that the reference deserializer reads and discards — the generator
special-cases it (see the comment in `generate_nodes.rb`).

## 2. Runtime (`Src/SuperIronRuby/Runtime`)

The object model, all in namespace `SuperIronRuby.Runtime`.

- **Value mapping** — Ruby values are plain CLR objects: `null`=nil, `bool`,
  `long`/`BigInteger`=Integer, `double`=Float, `MutableString`, `RubySymbol`
  (interned), `RubyArray : List<object?>`, `RubyHash` (insertion-ordered, Ruby
  `eql?`/`hash` keys), `RubyRange`, `RubyRegexp`, `RubyProc`, `RubyObject`,
  `RubyClass`/`RubyModule`. See `Docs/RUNTIME_NOTES.md` for the full table.
- **`RubyModule` / `RubyClass`** — method/constant/cvar tables, include/prepend,
  and MRI-style **ancestor linearization** (`[prepends ++ self ++ includes ++
  superclass-chain]`, deduped), cached and invalidated by a global version
  counter.
- **`RubyContext`** — the per-engine world: bootstraps the core class hierarchy
  (`BasicObject ← Object ← Module ← Class`, Kernel mixed into Object, Numeric/
  String/Array/...), `GetClassOf`, symbol interning, globals, the full exception
  hierarchy (`Exceptions.cs`), and **method dispatch** (`Dispatch.cs`): `Send`
  with a `(class, name)` cache, `method_missing`, visibility, and arity checks.
- **`BuiltinLoader`** — reflects over `[RubyClass]`/`[RubyModule]` types and binds
  their `[RubyMethod]` static methods (IronRuby style).
- **`ClrInterop.cs`** — `System::`-namespace resolution and reflection-based
  dispatch onto real CLR types.

## 3. Interpreter (`Src/SuperIronRuby/Interpreter`)

A tree-walking evaluator, a `partial class Interpreter` split by node family:
`Interpreter.cs` (driver + the central `Eval(Node)` switch), `.Literals`,
`.Calls`, `.Definitions`, `.Classes`, `.ControlFlow`, `.Assignments`, `.Strings`,
`.Blocks`, `.Patterns`. `RubyScope` is the lexical scope (locals, self, the
definition target, the current block, the break target). Non-local control flow
(`break`/`next`/`redo`/`return`/`retry`) uses the unwind exceptions in
`Runtime/Unwinds.cs`. Ruby-defined methods run through `RubyContext.InterpretedInvoker`,
installed by the interpreter at startup, so the runtime has no compile-time
dependency on the interpreter.

## 4. Builtin library (`Src/SuperIronRuby/Builtins`, `Lib/core/*.rb`)

Most core methods are C# (`[RubyMethod]`). Enumerable and Comparable are written
in **Ruby** (`Lib/core/30_enumerable.rb`, `20_comparable.rb`) on top of `each`/
`<=>` — embedded as resources and evaluated at engine startup, dogfooding the
interpreter.

### Adding a builtin method (worked example)

```csharp
[RubyClass("Array")]
public static class ArrayPrintOps
{
    [RubyMethod("sample")]                       // Array#sample
    public static object? Sample(RubyContext c, object? self, object?[] args, RubyProc? block)
    {
        var arr = (RubyArray)self!;
        return arr.Count == 0 ? null : arr[0];   // (deterministic stub)
    }
}
```

The four-parameter signature `(RubyContext, object? self, object?[] args,
RubyProc? block) -> object?` is mandatory; the loader binds it automatically.
Use `RubyMethodKind.Static` for class methods, `ModuleFunction` for
module-functions, and `Extends = typeof(T)` to route a custom CLR type to a Ruby
class.

## 5. Hosting & CLI

- **`Hosting/Ruby.cs`** — `Ruby.CreateEngine()` → `ScriptEngine` with
  `Execute`/`ExecuteFile`/`ExecuteToString`, a persistent top-level scope, and
  embedded-corelib loading.
- **`Src/SuperIronRuby.Console`** — the `sir` CLI (file / `-e` / REPL / `--ast`),
  MRI-style uncaught-exception formatting.

## Testing

- **xUnit** (`Tests/SuperIronRuby.Tests`) — 287 tests across parser, runtime, and
  every interpreter node family, each verified against ruby 4.0.2 semantics.
- **Differential** (`Tests/diff/run_diff.sh`) — runs snippets under both CRuby
  4.0.2 and `sir` and compares stdout byte-for-byte.
- **CLI smoke** (`Tools/smoke_cli.sh`).
